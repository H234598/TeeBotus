# Privacy and Encryption

TeeBotus is moving from Telegram sender memory to account-scoped memory. The privacy boundary is now the TeeBotus account ID inside one instance.

## Account ID and Communication Identities

- The account ID is a 128-character SHA-512 hex value generated from random bytes.
- The account ID is not a password, but the bot does not print it in group chats.
- Telegram user IDs, Signal UUIDs, Signal numbers, and future channels are stored as communication identities linked to an account.
- Linking a new communication identity requires account ID plus account secret.

## Account Secret

- The account secret is generated by `/register` and shown once in private chat.
- The plaintext secret is not stored.
- TeeBotus stores only an HMAC-SHA512 verifier.
- The HMAC pepper comes from per-instance Secret Service storage via `secret-tool`.
- `/rotate_secret` invalidates the previous secret.

## Encrypted Files

Structured account files are encrypted with AES-256-GCM. Keys are instance-scoped and stored in Secret Service.

Identity and account mapping use:

```text
account-identity-mapping-key
```

Structured account memory uses:

```text
account-structured-memory-key
```

Encrypted structured memory files:

- `User_Memory_Index.json`
- `User_Memory_Entries.jsonl`
- `OpenAI_State.json`

Account index, identity mapping, secret verifiers, profiles, and tombstones are encrypted as well.

## Plaintext Markdown

`User_Habbits_and_behave.md` intentionally remains plaintext. It is a manually editable Markdown note. It is merged during account merges, but it is not encrypted.

## What Is Protected

Disk-only access sees ciphertext for structured files. Disk-only access still sees Markdown notes in plaintext. Anyone with access to the running process, Secret Service, passphrase, or debugger can still reach plaintext because the bot must decrypt data to operate.
